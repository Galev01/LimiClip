# LimiClip v1.0 Roadmap — Design Spec

**Date:** 2026-06-08
**Author:** brainstormed with Claude Code
**Current version:** v0.4.1 (build 5)
**Target:** v1.0 — a stable, polished, world-distributable, feature-complete macOS clipboard manager

---

## 0. Context & Framing

LimiClip is a **local, single-user macOS menu-bar clipboard manager** (Swift 6, SwiftUI + AppKit, GRDB SQLite, KeyboardShortcuts, XcodeGen). It already ships: encryption at rest (AES-256-GCM via `FieldCipher`), a notarized + stapled DMG on GitHub, performance caps (history 100, max 5 images, max 15 pins), image caching, blob garbage collection, app exclusions, pause/incognito, quick actions, and screenshot import.

The original request — "fully scalable, deployable, worldwide easy install, security approved, feature rich, great UI/UX, no bugs/leaks, no CPU/RAM overflow" — maps onto a **local desktop app** as follows:

| Aspiration | What it actually means here | Status |
|---|---|---|
| Scalable | Stays fast & memory-bounded as history grows | Mostly done (caps + caching); harden in Cycle 1 |
| Deployable / worldwide / easy install | Auto-update + Homebrew + localization | Open → Cycle 3 |
| Security approved | Encryption + notarization + hardened runtime | Largely done; residual items |
| No bugs / leaks / CPU-RAM overflow | Stability + leak + edge-case hardening | Open → Cycle 1 |
| Great UI/UX | Onboarding, accessibility, polish | Open → Cycle 2 |
| Feature rich | Snippets, better search, sync | Open → Cycle 4 |

This is **not one project**. It is four sequenced cycles, each its own spec → plan → build → verify loop. This document specs all four in full detail.

## 1. Execution Strategy (dependency-aware parallelism)

The cycles are **not file-independent**, so they cannot all run simultaneously without merge conflicts:

- Cycle 1 (Stability) edits `DrawerView`, `ClipboardCard`, `DrawerWindowController`, `CompactPopupWindowController`, `ScreenshotImporter`, `FieldCipher`, `PasteInjector`, `RetentionJob`, `ImageCache`.
- Cycle 2 (UX) edits many of the **same** UI files (`DrawerView`, `ClipboardCard`, `*Pane`, empty states).
- Cycle 3 localization rewrites **every user-facing string** — i.e. the strings Cycle 2 is actively changing.
- Cycle 4 (Features) edits the store/schema and the same UI surfaces.

**Therefore the order is fixed:**

```
Cycle 1 (Stability)  ──►  Cycle 2 (UX)  ──►  Cycle 3b (Localization)
                     └─►  Cycle 3a (Sparkle + Homebrew, file-disjoint) ──┘ may run alongside Cycle 2
                                                          Cycle 4 (Features) ──► after Cycle 1, ideally after Cycle 2
```

**Safe parallelism = *within* a cycle, across disjoint files.** Each cycle's plan fans out subagents per-file or per-component where they don't touch the same file. Cross-cycle parallelism is limited to the file-disjoint pair (Cycle 2 UX ‖ Cycle 3a Sparkle/Homebrew), and only after Cycle 1 lands on `main`.

**Branching:** each cycle on its own branch off `main`; merge when green; next cycle branches from the merged result. Subagents within a cycle use `isolation: worktree` only when they would otherwise write the same file concurrently.

**Discipline (applies to every cycle):** TDD — failing test first, then fix; full suite green before merge; the user verifies by running the app. No commit/push/release without explicit user go-ahead (per user profile).

## 2. Cycle 1 — Stability, Leaks & Bugfixes

**Goal:** eliminate memory leaks, perpetual-CPU timers, crash paths, and the known/suspected bugs. This is the "no bugs / no leaks / no CPU-RAM overflow" cycle and the foundation everything else builds on.

All root causes below are **suspected from a static audit** and must be **confirmed with a failing test (or instrument) before the fix**, using `superpowers:systematic-debugging` where the cause isn't obvious.

### 2.1 Leaks & perpetual resources (HIGH)

| ID | File:line | Symptom | Fix approach |
|---|---|---|---|
| S1 | `UI/Drawer/DrawerView.swift:88` | `Timer.publish(every:1.0…).autoconnect()` subscribed via `.onReceive` with no teardown → fires forever, retains | Cancel on `.onDisappear`; store subscription in `@State`; or drive refresh from a store change-publisher instead of a wall-clock tick |
| S2 | `Services/ScreenshotImporter.swift` (~76–96) | Selector-based `NotificationCenter` observer + `NSMetadataQuery` started, no `deinit` to stop → leak + zombie query | Add `deinit { stop() }`; assign `query` before `start()`; prefer block-based observer with `[weak self]` |
| S3 | `UI/Drawer/DrawerWindowController.swift:36-39,69-79` & `UI/Compact/CompactPopupWindowController.swift:42,50-59` | Global `NSEvent` monitor + selector observer with no `deinit` cleanup; `NSScreen.screens[0]` force-subscript | Add `deinit` removing monitor + observer; replace `screens[0]` with `NSScreen.main ?? NSScreen.screens.first` guard |
| S4 | `Services/RetentionJob.swift:24-32` | Double-`start()` stacks concurrent timers | Guard `timer == nil` before scheduling (match `PasteboardMonitor` pattern) |

**Verification:** unit test that a `ScreenshotImporter`/window-controller `deinit` runs (weak-ref assertion); confirm no 1 Hz tick after drawer dismiss (no `onReceive` firing). Leaks/Instruments pass: launch, open/close drawer 50×, capture 50 items — RSS returns to baseline, no growing allocation.

### 2.2 Crash paths (HIGH/MED)

| ID | File:line | Symptom | Fix approach |
|---|---|---|---|
| S5 | `Services/PasteboardMonitor.swift:47-49` | `try!` fallback `BlobStore` crashes on disk-full / perms | Propagate error or use a guaranteed-safe fallback; never `try!` |
| S6 | `ActionsKit/QuickActionDetector.swift:97` | `parts[0]/parts[1]` subscript after a guard — redundant but fragile | Use safe destructuring (`parts.first` / `parts.last`) |
| S7 | Preview-only `try!` (`DrawerView:236`, `PrivacyPane:115`, `PreferencesView:58`, `CompactPopupView:62`, `ClipboardCard:250`) | Crash if store init fails in previews (non-prod) | `try?` with a stub fallback |

### 2.3 Data-correctness bugs (HIGH/MED)

| ID | File:line | Symptom | Fix approach |
|---|---|---|---|
| B1 | `ClipboardManagerTests/PasteInjectorTests.swift:65` + `Services/PasteInjector.swift:39-43` | **Known failing test**: image paste-from-blob. Suspected: `pasteboard.writeObjects([NSImage])` registers TIFF but not `.png`, and/or the test's `BlobStore(cipher: nil)` can't read encrypted blobs | **Debug first.** Likely fix: after `writeObjects`, also `setData(pngData, forType:.png)`; fix test to use a cipher. Confirm real paste of a saved image works end-to-end |
| B2 | `Store/FieldCipher.swift:55-63,78-84` | On **decryption failure** of sealed data, `open()` returns the ciphertext (magic header intact) → corrupt image / garbage text, silently | Distinguish "plaintext passthrough" (no magic prefix → fine) from "sealed-but-failed-to-open" (has prefix, open failed → log warning, return empty/throw, never raw ciphertext) |
| B3 | `Services/PasteInjector.swift:33-34` | Missing blob file referenced by DB row → unhandled throw on paste | Graceful fallback: `try?` → if nil, surface a non-fatal message; don't break the paste flow |
| B4 | `Store/ClipboardStore.swift:121` | Migration of file row with malformed JSON body silently hashes the raw body | Log + preserve prior `contentHash` on parse failure |

### 2.4 Edge-case hardening (LOW/MED)

- **Image byte-size cap** (`ClipboardStore.swift:33`): add a per-image byte cap (e.g. 10 MB) alongside the count cap, dropping oversized captures gracefully.
- **Disk-full on blob write** (`BlobStore.swift:70`): already throws + logged by callers; add a user-visible "storage full" signal (non-blocking).
- **Concurrent rapid copies / empty pasteboard / file-attr read failure:** audit confirmed these are handled — add regression tests so they stay handled.

### 2.5 Concurrency hygiene (LOW)

- `UI/ImageCache.swift:11` `@unchecked Sendable`: document the NSCache thread-safety contract or wrap access in a lock; move the decode-on-miss off the main thread (decode in a background `Task`, publish to cache).
- `nonisolated(unsafe)` observer tokens (`ClipboardViewModel:33`, `ExclusionsViewModel:14`, `AppCoordinator:20`): confirm init/deinit are both `@MainActor`; document the contract or make the property `@MainActor`-isolated.

### 2.6 Cycle 1 done criteria
- Every S/B/edge item above has a regression test (failing-first) and a fix.
- `PasteInjectorTests.testWritesImageFromBlob` passes; saved-image paste verified end-to-end in the running app.
- Instruments Leaks run clean across the open/close/capture stress loop; no perpetual timer.
- Full suite green. User runs the app and confirms.

## 3. Cycle 2 — UX Polish & Onboarding

**Goal:** "great UI/UX." Biggest gaps: **no onboarding** and **near-zero accessibility**.

### 3.1 First-launch onboarding (none exists today)
- New `OnboardingWindow`/flow gated by an `@AppStorage("didOnboard")` flag, shown from `AppCoordinator`/`AppDelegate` on first launch.
- Content: what LimiClip does (auto-captures clipboard), the core hotkey (⌘⇧V), the permission model (**Accessibility** for auto-paste; **screenshot-folder** watch), and privacy posture (local-only, encrypted).
- **Permission preamble** before the OS prompt, explaining *why* each permission is needed. Today Accessibility is only surfaced as a banner *inside* the drawer (`DrawerView:185-211`) after a failed paste.
- "What's New" sheet on major-version bump (last-seen-version check).

### 3.2 Accessibility (currently critical gap — zero support)
- **VoiceOver labels/hints/values** on every interactive element: `ClipboardCard`, `CompactClipboardCard`, `DrawerTabBar` tabs, search field, all buttons. E.g. card → `accessibilityLabel("Text: <preview>")`, `accessibilityHint("Return to paste, context menu for more")`.
- **Dynamic Type:** replace hardcoded `Font.system(size:)` in `DesignTypography.swift` with relative/scaled fonts; add `minimumScaleFactor` to truncated text; use `@ScaledMetric` for text-coupled spacing.
- **Reduce Motion:** gate the drawer slide (`DrawerWindowController:81-86`), hover-preview fade (`DrawerView:96`), and compact animations on `@Environment(\.accessibilityReduceMotion)`.
- **Keyboard completeness:** Tab focus cycling via `@FocusState` (Tab currently does nothing); make context-menu actions reachable by keyboard. (Arrows / Enter / Delete / ⌘1-9 / Esc already work.)

### 3.3 Discoverability & polish
- **Help/About menu**: shortcut legend (⌘⇧V, ⌘⇧A, arrows, Enter, Delete, ⌘1-9, Esc), permission guide, feature explanations.
- **Clarify the two screenshot toggles** (confusingly named): rename to "Save in-app screenshots (⌘⇧A)" and "Import system screenshots (⌘⇧4)" with a one-line footer that they're independent.
- **Default keybinding for the compact popup** (currently unset → users never find it).
- **Hover feedback on drawer cards** (`ClipboardCard` has none; match `CompactClipboardCard`); tooltips ("Click to paste").
- **Richer empty states** (`EmptyState.swift`) with a "Learn more" link to Help.
- **Per-pane explanatory headers** in Preferences; **app icons** beside excluded apps in `PrivacyPane` (`NSWorkspace.shared.icon(forFile:)`).
- **Adaptive card sizing** (replace hardcoded `184×210` / `340×240` with min/max frames) so the UI survives Dynamic Type and resize.

### 3.4 Cycle 2 done criteria
- Onboarding shows once on a fresh profile; permissions explained before prompts.
- VoiceOver can navigate and operate the drawer; Dynamic Type up to XXL doesn't clip; reduce-motion disables animations.
- Help/About reachable from the menu bar. User verifies UX in the running app.

## 4. Cycle 3 — Distribution Reach ("worldwide, easy install")

The release pipeline (`scripts/build-release.sh`: archive → export → notarize → staple → DMG → sign+notarize DMG, with `ExportOptions.plist`, `DISTRIBUTION.md`) is already solid. The gaps are reach.

### 4a. Auto-update — Sparkle *(file-disjoint; may run alongside Cycle 2)*
- Add **Sparkle** via SPM in `project.yml`.
- Generate an **EdDSA** key pair; sign each release DMG with Sparkle's `sign_update`.
- Publish an **appcast.xml** (GitHub Pages or release asset) with version, notes, DMG URL, signature.
- Wire `SPUStandardUpdaterController` in `AppCoordinator`; add `Info.plist` keys (`SUFeedURL`, `SUEnableAutomaticChecks`, `SUPublicEDKey`).
- Add "Check for Updates…" to the menu; extend `build-release.sh` to emit the appcast entry + signature automatically.
- Update `SECURITY.md` (which currently states "no auto-update yet").

### 4b. Homebrew cask *(file-disjoint)*
- Create `Casks/limiclip.rb` (version, sha256, GitHub release DMG URL, `app "LimiClip.app"`, `zap` for app-support dir).
- Document the per-release bump (version + SHA) and ideally script it from `build-release.sh`.

### 4c. Localization *(must run AFTER Cycle 2 — UX changes the strings)*
- Migrate ~30–40 hardcoded `Text("…")` strings to a **String Catalog** (`Localizable.xcstrings`); use `LocalizedStringKey`.
- Handle plurals ("1 item" / "N items").
- Ship **English + 2–3 launch languages** (e.g. ES, FR, DE). `KeyboardShortcuts` already bundles many locale strings.

### 4.4 Cycle 3 done criteria
- A built DMG triggers an in-app update from a prior version against the live appcast (signature verified).
- `brew install --cask limiclip` installs the notarized app (tap or core, per user's choice).
- UI renders correctly in each shipped language; no clipped/untranslated strings.

## 5. Cycle 4 — Feature Richness

Runs on the stabilized, polished base (after Cycle 1; ideally after Cycle 2).

### 5.1 Snippet library *(highest value; schema partly ready)*
- `Item.snippetId: Int64?` FK already exists. Add a `snippets` table (id, name, keyword, body, createdAt).
- Promote a pinned item → named snippet; manage in a new Snippets pane.
- **Keyword expansion** on paste (type a keyword → expand to snippet body), respecting exclusions/secure fields.

### 5.2 Search improvements
- **Fuzzy** matching and optional **regex** mode in the drawer/popup search (SQLite `REGEXP` or in-memory fuzzy ranker), with clear UI affordance.

### 5.3 Deferred / demand-gated
- **iCloud sync** (CloudKit + conflict resolution + encryption-key story) — large; defer until there's demand. Spec separately when prioritized.
- Content-based secret heuristics (auto-conceal detected secrets) — deferred from the security audit.

### 5.4 Cycle 4 done criteria
- Create/expand a snippet end-to-end; expansion honors exclusions. Fuzzy/regex search returns expected results. Full suite green; user verifies.

## 6. Cross-cutting quality bar (every cycle)
- **TDD**: failing test first; no fix without a test that reproduces the issue.
- **No regressions**: full suite green before merge (baseline ~190 tests).
- **Performance budget**: idle CPU ≈ 0 when drawer closed; bounded RSS under the stress loop; no main-thread stalls > 1 frame.
- **Security**: local-only, no telemetry/network (preserve); encryption invariants intact; no secrets in repo.
- **User verification**: the user runs the app and confirms each cycle before release; no commit/push/release without explicit go-ahead.
