# Security Policy

## Reporting a vulnerability

If you find a security or privacy issue in LimiClip, please report it privately:

- Open a **GitHub private security advisory**: <https://github.com/Galev01/LimiClip/security/advisories/new>, or
- Email the maintainer (see the GitHub profile) with the subject `LimiClip security`.

Please do **not** open a public issue for security problems. We aim to acknowledge
reports within a few days. There is no bug-bounty program; this is a personal project.

## How LimiClip handles your data

LimiClip is a local macOS clipboard-history manager. Its security model:

- **Local only.** Clipboard data never leaves your Mac. There is no network code,
  telemetry, analytics, or crash reporting anywhere in the app.
- **Encrypted at rest.** Clipboard contents (text, file paths, source-app names)
  and image thumbnails are encrypted with **AES-256-GCM** (Apple CryptoKit). The
  key is generated with a CSPRNG and stored in your **Keychain**
  (`kSecAttrAccessibleWhenUnlockedThisDeviceOnly` — not synced, not exportable to
  another device). The deduplication hash is a keyed HMAC-SHA256, so short copied
  secrets can't be brute-forced from the database. Deleting an item removes its
  blob immediately, and "Clear All" runs `secure_delete` + `VACUUM`.
- **Sensitive copies are skipped.** Copies that apps mark concealed/transient
  (`org.nspasteboard.ConcealedType`, `TransientType`, `AutoGeneratedType`,
  `Pasteboard generator type`) are never recorded — this covers cooperating
  password managers (1Password, Bitwarden, KeePassXC, …).
- **App exclusions.** Major password managers, Keychain Access, and the macOS
  Passwords app are excluded by default; you can add more in Preferences. An
  optional "strict capture" mode skips copies whose source app can't be identified.
- **Screenshots are opt-in.** The ⌘⇧A screenshot is copied to the clipboard but
  **not** saved to history unless you enable "Save screenshots to history".
- **No elevated privileges.** The app uses Accessibility (to synthesize ⌘V for
  paste) and no network entitlements. It is Developer ID signed, Hardened-Runtime
  enabled, and notarized by Apple.

### Known limitations

- **Browser / iCloud-Keychain passwords.** Passwords copied via browser AutoFill
  or the iCloud Keychain popover arrive under the browser's or system-UI bundle id
  (not the password manager's), so they can't be excluded by app. Rely on apps
  marking such copies concealed, or use strict-capture mode.
- **No auto-update yet.** Update by downloading the latest signed release from
  GitHub. Verify the published SHA-256 of the DMG before installing.
- **FileVault recommended.** At-rest encryption protects the database and blobs,
  but enabling FileVault is still recommended for full-disk protection.

## Supported versions

Only the latest released version receives security fixes.
